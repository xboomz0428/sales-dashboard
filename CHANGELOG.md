# 更新日誌 CHANGELOG

## v0.0.79（2026-07-07）

### 修正：年度銷售金額大幅短少（可能只剩實際的不到一成）

**問題根源（三個疊加）：**
1. 資料庫 `sales_data` 資料表從未被建立，程式讀/寫 DB 一律失敗，只能退回讀 Storage 的 `.rows.json` 快取。
2. 大型 `.xls` 檔在瀏覽器（尤其手機）解析時記憶體不足而中途失敗，只把「解析到一半的殘缺結果」存進快取，之後每次載入都讀這份殘缺資料。
3. `excelCore` 的檔案內去重把「同訂單、同商品、同數量、同金額」的**正當重複明細**誤判為重複而刪除。

**本次修正：**
- 建立 `sales_data` 資料表（含 `source_file`、`import_batch` 欄位與寫入權限政策），並以伺服器端解析一次灌入正確完整的歷史資料。DB 為主要資料來源，載入不再依賴瀏覽器解析。
- `excelCore.js`：移除會誤刪正當重複明細的去重；`_key` 加上檔案內行序號，確保每一筆銷售明細都保留（銷售金額為已發生事實，全部統計）。
- `useCloudData.js`：上傳改為**檔案層級冪等 + 寫入後驗證筆數**——先以新批次插入、驗證筆數相符後才刪除舊批次；插入失敗會還原並**明確報錯**，不再出現「✓ 已同步（DB 失敗）」這種假成功。
- `useCloudData.js`：DB 分頁讀取加入穩定的次要排序（date, id），避免同日期資料列在分頁時被跳過或重複。
- `App.jsx`：上傳時傳整份檔案的資料列給資料庫（而非去重後的差集），確保檔案層級取代不遺漏任何一筆。

### 其他
- 統一版本號顯示（登入頁 v0.0.24 → v0.0.79，與 package.json 同步），新增本 CHANGELOG。
